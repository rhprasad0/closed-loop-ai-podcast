# CLAUDE.md

Project context and conventions for AI assistants working on this codebase.

## What This Is

"0 Stars, 10/10" is a closed-loop multi-agent podcast pipeline. AI agents run weekly on AWS to find underrated GitHub projects, write comedy podcast scripts about them, generate cover art, produce audio, and publish episodes — fully autonomous, no human in the loop.

The infrastructure is AWS serverless: Step Functions orchestrates Lambda functions, Bedrock provides model access (Claude for reasoning, Nova Canvas for image generation), ElevenLabs handles TTS, and everything is Terraformed.

This repo is a portfolio project demonstrating production agentic AI engineering.

## Key Architecture Decisions

**Step Functions (Standard) orchestrates the pipeline.** Standard, not Express — the pipeline takes several minutes end-to-end and we need exactly-once execution, full audit trail, and the evaluator-optimizer retry loop via Choice states.

**One Lambda per agent.** Separate functions keep IAM permissions tight (Discovery needs Exa, TTS needs ElevenLabs, Cover Art needs Bedrock image access, Post-Production needs the ffmpeg layer) and allow independent iteration. Each Lambda's deployment package includes its prompt files — no external prompt storage.

**Prompts live in Git, deploy with code.** Each Lambda reads its system prompt from its own deployment package at runtime via `LAMBDA_TASK_ROOT`:
```python
import os
prompt_path = os.path.join(os.environ['LAMBDA_TASK_ROOT'], 'prompts', 'discovery.md')
with open(prompt_path) as f:
    system_prompt = f.read()
```
Terraform's `archive_file` data source packages handler + prompts together. Prompt changes trigger redeployment via `source_code_hash`.

**Evaluator-optimizer loop for script quality.** The Producer Lambda evaluates the Script Lambda's output and returns PASS/FAIL with structured feedback. On FAIL, a Step Functions Choice state routes back to the Script Lambda with the feedback appended to its input. Max 3 attempts before the execution errors out. This is implemented in ASL in `terraform/step-functions.tf`.

**Two agents read from Postgres for cross-episode learning:**
- **Discovery agent** queries `episode_metrics` (LinkedIn performance) and `featured_developers` (dedup) to inform search objectives. Episodes about project types that performed well bias future searches.
- **Producer agent** queries top-performing episode scripts (by LinkedIn metrics) to use as quality benchmarks alongside its static rubric.

**Cover art generated by Nova Canvas on Bedrock.** The Cover Art Lambda generates episode artwork using Amazon Nova Canvas (`amazon.nova-canvas-v1:0`). This keeps image generation in the AWS ecosystem — no external API key needed, same Bedrock client as the other agents. The prompt is generated by the Script or Producer agent based on the episode content.

**Dynamic website, not static generation.** The podcast site is a Lambda Function URL that queries the `episodes` table and renders HTML via Jinja2 templates, fronted by CloudFront with a ~1 hour TTL. New episodes appear automatically when the pipeline writes to Postgres. This decouples the site from the pipeline — a site issue doesn't fail the Step Functions execution, and the site can show live data (current GitHub star counts, engagement metrics) that a static build can't.

## Tech Stack

- **Language:** Python 3.12 (all Lambdas)
- **IaC:** Terraform
- **Models:** Claude via AWS Bedrock (tool use for Discovery and Research agents)
- **Image Gen:** Amazon Nova Canvas via AWS Bedrock
- **TTS:** ElevenLabs `/v1/text-to-dialogue` API, `eleven_v3` model
- **Database:** PostgreSQL (RDS)
- **Media:** ffmpeg Lambda Layer for audio → video conversion

## The Pipeline

```
EventBridge (weekly cron)
  → Discovery Lambda (Bedrock + Exa API, reads episode history + metrics from Postgres)
  → Research Lambda (Bedrock + GitHub API)
  → Script Lambda (Bedrock, writes 3-persona comedy script)
  → Producer Lambda (Bedrock, reads top scripts from Postgres, evaluates → PASS/FAIL)
      └─ FAIL? → back to Script Lambda (max 3 retries, with structured feedback)
  → Cover Art Lambda (Bedrock Nova Canvas → PNG to S3)
  → TTS Lambda (ElevenLabs API → MP3 to S3)
  → Post-Production Lambda (ffmpeg → MP4, writes episode record to RDS, assets to S3)
```

The website (`lambdas/site/`) is a separate Lambda Function URL, not part of the pipeline. It reads from Postgres + S3 on every request.

## Podcast Specifics

Three AI personas: **Hype** (optimist), **Roast** (British cynic), **Phil** (philosopher). Each episode follows a six-segment arc: intro, core debate, developer deep-dive, technical appreciation, hiring manager segment, outro with callbacks.

**Hard constraint:** ElevenLabs text-to-dialogue API has a 5,000 character limit. Scripts must target 4,000–4,500 characters. The Producer agent enforces this.

## Database Tables

- **`episodes`** — Episode catalog. Powers both the pipeline and the website. Columns: episode_id, air_date, repo_url, repo_name, developer_github, developer_name, star_count_at_recording, script_text, research_json (jsonb), cover_art_prompt, s3_mp3_path, s3_mp4_path, s3_cover_art_path, producer_attempts, created_at.
- **`episode_metrics`** — LinkedIn performance snapshots. Read by Discovery agent for search objective tuning and Producer agent for quality benchmarking. Columns: metric_id, episode_id (FK), linkedin_post_url, views, likes, comments, shares, snapshot_date.
- **`featured_developers`** — Dedup list. Read by Discovery agent to avoid re-featuring. Columns: developer_github (PK), episode_id (FK), featured_date.

## Terraform Conventions

- All resources are in `terraform/`. No modules — every Lambda defined inline for simplicity.
- Secrets (API keys) are in Secrets Manager, referenced by Lambda environment variables.
- The Step Functions state machine ASL definition lives in `terraform/step-functions.tf` as a `jsonencode()` block, not a separate JSON file.
- The website Lambda Function URL and CloudFront distribution are in `terraform/site.tf`.
- Tags: `project = "0-stars-podcast"`, `managed_by = "terraform"` on all resources.

## Lambda Conventions

- Python 3.12 runtime, all functions.
- Handler pattern: `handler.lambda_handler(event, context)`.
- Shared utilities (Bedrock client, DB connection, S3 helpers) are in the `shared/` Lambda Layer, imported as `from shared import bedrock, db, s3`.
- Each function's `prompts/` directory contains markdown prompt files bundled into the deployment ZIP.
- State is passed between Lambdas via the Step Functions state object (JSON). Each Lambda receives the accumulated output of all previous steps and adds its own output.
- Error handling: Lambdas raise specific exception classes. Step Functions Retry/Catch blocks handle transient errors (Bedrock throttling, API timeouts) with exponential backoff. Business logic failures (Producer FAIL verdict) are returned as normal output and handled by Choice states.
- The site Lambda (`lambdas/site/`) is NOT part of the Step Functions pipeline. It runs independently behind a Function URL, reads from Postgres, renders Jinja2 templates, and returns HTML.

## Writing and Content Conventions

When working on prompts, scripts, or any text content for this project:

- **No AI slop vocabulary.** No "delve," "landscape," "leverage," "at its core," "it's not just X — it's Y," "game-changer," "groundbreaking." If it sounds like ChatGPT wrote it, rewrite it.
- **No promotional language.** No "thrilled to share," "excited to announce," "proud to present."
- **Specific over general.** Always prefer a concrete technical detail over a vague claim.
- **The podcast is an "agent system" or "multi-agent workflow."** Not "an AI podcast." This is a portfolio piece.
